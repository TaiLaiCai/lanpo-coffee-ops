# 蓝珀咖啡 · 飞书多维表格导入包

8 张 CSV，UTF-8 + BOM 编码，飞书多维表格直接导入。

## 一、导入步骤

1. 飞书云文档 → 新建多维表格 → 命名"蓝珀咖啡运营底座"
2. 表格内点 `+` 新建数据表 → 选「导入数据」→ 选「CSV」→ 上传对应文件
3. 字段类型多数会自动识别，**注意手动改这几类**：
   - **日期字段** 若识别成"文本"，改成「日期」
   - **金额字段**（营业额/成本/GMV）改成「数字 - 货币」
   - **多选字段**（投放渠道/客户标签/适合场景/杯型）改成「多选」并配置选项
   - **单选字段**（天气/平台/状态等）改成「单选」并配色
   - **人员字段**（在班人员/负责人/处理人）改成「人员」类型
   - **复选框字段**（是否节假日/店长签字/是否爆款/是否含奶）改成「复选框」

## 二、必须手动配置的公式字段

CSV 里这些字段已填了示例数值，**导入后改成公式字段**，删掉示例数值让它自动算：

| 表 | 字段 | 公式 |
|---|---|---|
| 1 经营日报 | 星期 | `TEXT([日期],"dddd")` |
| 1 经营日报 | 堂食客单价 | `[堂食营业额]/[堂食杯量]` |
| 1 经营日报 | 进店转化率 | `[堂食杯量]/[进店人数]` |
| 2 外卖日报 | 实收 | `[GMV]-[平台佣金]` |
| 2 外卖日报 | 实收占比 | `[实收]/[GMV]` |
| 2 外卖日报 | 客单价 | `[GMV]/[单量]` |
| 3 内容数据 | 收藏率 | `[收藏]/[72h曝光]` |
| 3 内容数据 | 是否爆款 | `IF([72h曝光]>10000,"是","否")` |
| 4 活动 | ROI | `[带来GMV]/[实际成本]` |
| 5 客诉 | 闭环时长 | 视实际工时字段而定，可手填 |
| 6 库存 | 理论用量 | `[期初库存]+[入库]-[期末库存]` |
| 6 库存 | 损耗 | `[理论用量]-[实际用量]` |
| 7 会员 | 沉睡天数 | `TODAY()-[最近到店]` |
| 8 菜单 | 毛利率 | `([售价]-[物料成本])/[售价]` |

## 三、跨表关联（导入完所有表后再配）

- 表 3「关联产品」→ 关联到表 8「产品名」
- 表 4「关联产品」→ 关联到表 8「产品名」
- 表 7「偏好产品」→ 关联到表 8「产品名」

## 四、必装的 3 条自动化流程

在多维表格右上角「自动化」里配：

### 1. 闭店数据快照（每晚 22:00）
- 触发：定时
- 动作：把当日表 1 + 表 2 + 表 3 数据汇总，发到运营群
- （进阶）调用 webhook 喂给 Agent

### 2. 差评告警
- 触发：表 2「新增差评数」> 0
- 动作：在表 5 新建一行 + @ 店长

### 3. 会员唤醒
- 触发：每周一 10:00
- 动作：筛选表 7 中「沉睡天数 ≥ 30」且「累计消费 ≥ 200」的记录 → 发到店长

## 六、首周冷启动节奏

- **Day 1**：导入 8 张表，配字段类型 + 公式（约 90 分钟）
- **Day 2**：补完 8 表的视图（约 60 分钟）
- **Day 3-7**：开始每天填表 1 / 表 2 / 表 3，跑通闭环
- **Day 8 起**：在飞书自动化里配那 3 条流程

## 七、字段口径备注

- **堂食营业额**：仅含店内+自取，不含外卖
- **外卖 GMV**：平台显示的总流水（含平台补贴前）
- **平台佣金**：技服费 + 推广费 + 配送费分摊
- **新增差评数**：评分 ≤ 3 星的新评价
- **到店核销**（小红书）：通过笔记带券到店核销的实际杯数
- **沉睡天数**：以最近一次到店为准，外卖单量也算到店
